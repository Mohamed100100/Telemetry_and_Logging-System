### 1. Thread-Safe Ring Buffer

A **Ring Buffer** (Circular Buffer) is a fixed-size data structure that wraps around. When full, it can either overwrite old data or reject new data.

#### Why Ring Buffer?

| Feature | Benefit |
|---------|---------|
| Fixed Memory | No dynamic allocation during operation |
| O(1) Operations | Constant time push/pop |
| Cache Friendly | Contiguous memory layout |
| Bounded | Prevents memory exhaustion |

### 2. Condition Variables

A **Condition Variable** allows threads to efficiently wait for a condition to become true, without wasting CPU cycles.

#### The Problem: Busy Waiting (Bad!)

```cpp
// BAD - Burns CPU doing nothing!
void worker() {
    while (true) {
        if (!buffer.empty()) {
            process(buffer.pop());
        }
        // If empty, immediately checks again... and again... 100% CPU!
    }
}
```

#### The Solution: Condition Variable (Good!)

```cpp
// GOOD - Worker sleeps when no work
void worker() {
    while (true) {
        std::unique_lock<std::mutex> lock(mutex);
        
        // Sleep until buffer has data (0% CPU while sleeping!)
        cv.wait(lock, []{ return !buffer.empty() || stopping; });
        
        // Woken up - either have data or need to stop
        if (stopping && buffer.empty()) return;
        
        auto item = buffer.pop();
        lock.unlock();
        
        process(item);
    }
}
```
